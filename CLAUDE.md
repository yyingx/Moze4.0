# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 概述

本仓库包含 Python 脚本，用于将支付宝、微信、云闪付账单转换为 Moze 4.0 导入格式，实现自动记账。

- **支付宝/微信**: 解析 CSV/XLSX 文件，基于规则字典和关键词智能分类
- **云闪付**: 使用 PaddleOCR 识别账单截图，智能提取交易信息

## 运行脚本

### 支付宝/微信账单转换

```bash
python Moze4_0_Import_v11_67.py
```

脚本会弹出文件选择对话框，选择账单文件后输出 CSV 到 `Moze4.0_Import/` 目录。

**依赖安装：**
```bash
pip install pandas numpy openpyxl
```

### 云闪付 OCR 识别（稳定版）

```bash
python "Moze4.0_云闪付PaddleOCR 稳定版_.py"
```

选择云闪付账单截图（支持多选），OCR 识别后输出到 `Moze4.0_Import/` 目录。

**环境要求：**
```bash
pip install paddlepaddle==2.6.2 paddleocr==2.7.3
pip install opencv-python pandas numpy
```

**版本**: 2026.01.22_Stable_v2
**配置**: 修改脚本中 `DEFAULT_YEAR`、`MY_NAME`、`ACCOUNT_MAP` 等参数

## 架构

### 主脚本 (`Moze4_0_Import_v11_66_8.py`)

处理流程：
1. `load_settings()` / `load_rules()` - 从 `Moze Dict.xlsx` 加载配置和规则
2. `sniff_and_load_data()` - 检测文件类型（CSV=支付宝，XLSX=微信）并解析
3. `process_transfers()` - 处理账户间转账交易
4. `process_main()` - 主交易处理，匹配字典规则
5. `process_heuristics()` - 基于 `DATA_SOURCE` 关键词的自动分类
6. `save_result()` - 输出带时间戳的 CSV

### 核心数据结构

**请勿随意修改以下结构，它们相互依赖：**

- `DATA_SOURCE` - 各类别的关键词列表（水果、蔬菜、肉类等）
- `INGREDIENT_PRIORITY` - 食材分类处理顺序：`(key, 名称, 子类别)`
- `RAW_MAPPING_CONFIG` - 子类别映射到 `(记录类型, 主类别, 项目)`
- `AUTO_MAP_DICT` - 由 `RAW_MAPPING_CONFIG` 派生，子类别 → 元组

`DATA_SOURCE` 和 `INGREDIENT_PRIORITY` 的顺序必须对应。

### 分类优先级

1. **备注驱动** - 用户在备注中标注，如 `探索claude` → 子类=探索，描述=claude
2. **字典规则** (`Moze Dict.xlsx`) - 先精确匹配商家名，再正则匹配
3. **关键词检测** (`DATA_SOURCE`) - 在商家/商品名中检测食材/商品关键词
4. **时间推导餐饮** - 检测为正餐但无子类别时，根据交易时间判断早餐/午餐/晚餐/夜宵

### 淘宝订单识别

通过商户单号前缀 `T200P` 识别淘宝订单（兼容 T200P4、T200P5 等）。

### 转账处理

通过 `CONFIG['TRANSFER_TARGET_x']` 配置转账对方，自动生成转入/转出配对记录。

### 手动增加行（v11.67 新增）

在支付宝/微信账单 Excel 中手动添加交易记录时，**可以不填"收/支"列**，系统会根据备注关键词自动推断：

| 备注格式 | 自动推断为 | 说明 |
|---------|----------|------|
| `借入xxx` | 收入 | 应付款项（收钱） |
| `借出xxx` | 支出 | 应收款项（借出钱） |
| `代付xxx` | 支出 | 应收款项（代付款） |
| `报账xxx` | 支出 | 应收款项（公司报销） |
| `押金xxx` | 支出 | 应收款项（押金） |
| `材料费xxx` | 支出 | 应收款项（费用报销） |
| `物流运输xxx` | 支出 | 应收款项（费用报销） |

**使用场景：** 现金交易、未在支付宝/微信记录的交易等。

### 应收应付款项逻辑

**应收款项**（记录类型 = `应收款项`）
- **子类别**: 报账、借出、代付、押金
- **金额**: 负数（支出）
- **对象**: 必填（借给谁、代付给谁）
- **商家/项目**: 自动清空
- **触发**: 备注以关键词开头 + 收/支=支出

**应付款项**（记录类型 = `应付款项`）
- **子类别**: 借入
- **金额**: 正数（收入）
- **对象**: 必填（从谁借入）
- **商家/项目**: 自动清空
- **触发**: 备注以"借入"开头 + 收/支=收入

**点号分隔格式**（v11.67 新增）：
- 在应收应付类别中，支持 `对象.描述` 格式
- 示例：`借入谢辉.钉子` → 对象=谢辉，描述=钉子

## 云闪付 OCR 特性

### 智能识别
- **日期推导**: 自动识别月-日格式，跨年时自动调整年份
- **金额提取**: 处理 OCR 误识别（l/I/| → 1），合并多段数字
- **账户映射**: 支持尾号匹配 + 特殊账户名（活期+、零钱等）
- **时间修正**: 自动修正常见 OCR 错误（如 71 → 11）

### 自动分类逻辑
- **转账识别**: "转给" + 本人姓名 → 转出/转入
- **借贷识别**: "转给/来自" + 他人姓名 → 借出/借入（应收/应付款项）
- **还款识别**: "还款" 关键词 → 信用卡还款

### 验证机制
1. **闭环验证**: 检查转账配对（金额相反、日期相同），自动同步时间
2. **时间检查**: 提示缺失时间的记录，建议补充
3. **对象清理**: 自动去除"转账"、"付款"等后缀，提取纯净人名

### 配置项

```python
EXCLUDE_RECEIVABLES = False   # 是否排除应收应付

```
